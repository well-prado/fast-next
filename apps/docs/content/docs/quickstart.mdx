## 1. Install & Bootstrap

```bash
pnpm install
pnpm dlx create-fast-next@latest init . \
  --install auto \
  --with-auth \
  --with-queue \
  --with-cache redis \
  --with-mcp \
  --with-docker \
  --orm prisma --db postgres
```

All flags are optional—omit them to answer prompts interactively. The CLI drops the API catch-all, Fastify server helpers, Better Auth bridge, optional queue/cache/MCP/Docker scaffolds, and (if selected) ORM boilerplate. After running with `--with-auth`, edit `src/server/services/auth/better-auth.ts` to configure adapters, secrets, and providers before hitting `/api/auth/*`. When you stay in interactive mode, the prompt mirrors the shadcn CLI: database → ORM → a multi-select of optional modules (↑/↓ + space, enter to confirm).

## 2. Run the Playground App

```bash
pnpm --filter fast-next-playground dev
```

Visit `http://localhost:3000`. The `/demo` page renders data fetched via:

- `api.system.health.request()` / `api.projects.list.request()` inside a **Server Component** (server caller, zero HTTP hop).
- `api.projects.list.useQuery()` and `api.projects.create.useMutation()` inside a **client component** (browser client hooks backed by `FastifyQueryClient`).

## 3. Smoke Test HTTP

```bash
curl http://localhost:3000/api/health
curl http://localhost:3000/api/users/1
```

These hit Fastify handlers defined in `apps/fast-next-playground/src/server/http/routes`.

## 4. Explore the Server Client

```ts
import { api, queryClient } from "@/server/api";

const health = await api.system.health.request();
const projects = await api.projects.list.request();
await queryClient.fetchQuery("projects", "list");
```

Every route automatically exposes `query`/`mutate` helpers plus query-client utilities for batching.

## 5. Try the React Hooks

In `ProjectsClientPanel`:

```ts
const query = api.projects.list.useQuery();
const mutation = api.projects.create.useMutation({
  invalidate: { resource: "projects" },
});
```

`useQuery` refetches automatically after invalidation, and `useMutation` exposes TanStack-like state flags.

## 6. Queue, Cache, MCP, Docker Commands

```bash
pnpm dlx create-fast-next queue start
pnpm dlx create-fast-next queue status --redis redis://localhost:6379
pnpm dlx create-fast-next cache stats
pnpm dlx create-fast-next mcp tool query-db
pnpm docker:up   # bring up app + postgres + redis + mcp
```

## 7. Type-Check & Lint

```bash
pnpm --filter @fast-next/fastify-zod-router check-types
pnpm --filter @fast-next/fastify-server-client check-types
pnpm --filter fast-next-playground check-types
pnpm lint
```

With these basics working you’re ready to dive into the architecture, CLI, and feature guides.
